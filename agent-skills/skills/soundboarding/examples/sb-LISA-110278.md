# Verify Builds and Agents with ACC Bamboo v12.1.6

> **Goal:** Validate that all Bamboo agents and build plans are compatible and operational after the PWC Bamboo ACC
> Environment upgrade to version 12.1.6, ensuring Java 21 compliance across all components.

---

## ⚠️ Java Version Requirement

Bamboo 12.1.x requires **Java 21 as a minimum** (Java 17 support has been dropped as of Bamboo 12.0).

| Java Version | Compatible with Bamboo 12.1.x? |
|---|---|
| Java 8 | ❌ Will NOT connect to Bamboo 12.1.x master |
| Java 11 | ❌ Will NOT connect to Bamboo 12.1.x master |
| Java 17 | ❌ Support dropped in Bamboo 12.0 — do NOT use |
| Java 21 | ✅ Required minimum (OpenJDK / Oracle JDK / Temurin) |
| Java 25 | ✅ Supported — installed as additional build capability |

> **👉 Action required for all System Teams:** Upgrade your agents to Java 21 no later than **May 15th, 2026**.

## Repository Changes Required

The following code changes have been identified and applied across `sara-bamboo-agent-deployment` and `sara-bambooagent-helm` to meet Bamboo 12.1.6 / Java 21 compliance. These must be reviewed, merged, and a new image must be built before the ACC deployment.

### `sara-bamboo-agent-deployment`

#### `docker/Dockerfile`

| # | File | Change | Reason |
|---|---|---|---|
| 1 | `docker/Dockerfile` | `BAMBOO_VERSION=9.6.9` → **`12.1.6`** | The agent bootstrap JAR downloaded at image build time must match the Bamboo master version. An agent built against 9.6.9 will not connect to a 12.1.x master. |

> **Already compliant ✅ (no change needed):**
> - Java 21 (`/home/cadappl_RHEL6/openjdk/21.0.2`) is set as the system default via `alternatives` and `JAVA_HOME`.
> - Temurin 25 JDK is installed at `/usr/lib/jvm/temurin-25-jdk` and registered as a build capability.
> - The wrapper installer is already invoked with the Temurin 25 JDK.

#### `docker/bamboo-wrapper.conf`

| # | Property | Before | After | Reason |
|---|---|---|---|---|
| 2 | `wrapper.java.version.min` | `8` | **`21`** | Bamboo 12.x requires Java 21 minimum. Leaving this at 8 would allow the wrapper to start with Java 8/11, which would then fail to connect. |
| 3 | `wrapper.java.version.fallback` | `11` | **`21`** | Fallback must not drop to Java 11 which is incompatible with Bamboo 12.x. |

> **Already compliant ✅ (no change needed):**
> - `wrapper.java.command=/usr/bin/java` correctly points to the Java 21 binary via the `alternatives` symlink established in the Dockerfile.
> - `--add-opens` JVM flags and `--enable-native-access=ALL-UNNAMED` are already present for Java 21 module compatibility.

#### `docker/bamboo-agent`

| # | Property | Before | After | Reason |
|---|---|---|---|---|
| 4 | `CADAPPL_JAVA` fallback | `/home/cadappl_RHEL6/jdk/1.8.0.172u/jdk/bin/java` | **`/home/cadappl_RHEL6/openjdk/21.0.2/openjdk/bin/java`** | If `/usr/bin/java` is missing at runtime, the previous fallback would start the agent wrapper with Java 8 — incompatible with Bamboo 12.x. |

### `sara-bambooagent-helm`

#### `values.yaml`

| # | Property | Change | Reason |
|---|---|---|---|
| 5 | `agent_server_url` | Added comment documenting the ACC URL: `https://apps-bamboo-pwc-acc.asml.com/agentServer` | The file currently only contains the production URL. ACC deployments must switch to the ACC server URL to avoid agents connecting to the production master during testing. |
| 6 | `imagetag` (all 4 pools) | Must be updated to the **new image tag** after rebuilding | All four pools (`bambooagent_a_pool`, `bambooagent_b_pool`, `bambooagentlight_a_pool`, `bambooagentlight_b_pool`) reference `2026-02-17_13-53` which is the old image built against Bamboo 9.6.9. |

> **Already compliant ✅ (no change needed):**
> - `bamboo-capabilities.properties` already defines `OpenJDK 21.0.2` and `OpenJDK 25.0.1_8_temurin` as build capabilities.
> - Older JDKs (1.8, 11, 17) remain registered as executable capabilities — build plans that require older Java versions can still use them.

## Notes

- **Linked Repository Configurations:** The Bamboo ACC environment may still be linked to BB production repositories (not BB ACC). You can only confirm the actual linked repo by inspecting the build logs. To fix the link, open the Linked Repository configuration and press **Save**.
- **Build Capabilities:** While the agent wrapper must run on Java 21, other JDKs (e.g., Java 8, 11, 17) can still be defined as **Executable Capabilities** for projects requiring older versions.
- **Basic Auth on REST APIs:** Basic Auth is disabled by default in new Bamboo instances but remains enabled when upgrading existing ones. Verify that all current API integrations continue to work as expected.
- **VCP Isolation:** Due to environment isolation, not all scenarios can be tested (e.g., contacting VCPs from ACC).
- **ACC vs PRD values:** Consider maintaining a separate `values-acc.yaml` override file (or using Helm `--set`) to avoid accidentally deploying ACC configuration to production.

## Version Change Summary

| Version Transition | Noticeable Change | Note |
|---|---|---|
| 9.6.9 → 10.0.0 | Dark and light theme support | — |
| 9.6.9 → 10.0.0 | REST v2 (rearchitected Java APIs) | Impact on Marketplace app developers only |
| 9.6.9 → 10.0.0 | New execution strategy for concurrent builds (block or stop oldest) | Worth investigating! |
| 9.6.9 → 10.0.0 | Build number modification for plans and plan branches | Useful for CI migrations |
| 10.0.0 → 10.1.0 | Java 21 support on server and agent sides | — |
| 10.1 → 11.0 | Automatic offline agents management (auto-removal policies) | — |
| 11.0 → 12.0.0 | Service Accounts (non-person accounts for services/integrations) | Verify if applicable for our setup |
| 11.0 → 12.0.0 | Non-rerunnable plans (only fresh builds allowed) | Consider enabling by default |
| 11.0 → 12.0.0 | **Java 21 minimum required — Java 17 support dropped** | ⚠️ Critical |
| 11.0 → 12.0.0 | Basic Auth disabled by default on REST APIs (new instances) | **Must verify our API calls still work!** |
| 12.0.0 → 12.1.0 | LTS version | — |
