# Verify Builds and Agents with ACC Bamboo v12.1.6

> **Goal:** Validate that all Bamboo agents and build plans are compatible and operational in ACC with version 12.1.6
---


## Acceptance Criteria

- [ ] A PR is prepared in which our Maven dependency is updated to use the new Bamboo API version
  - [ ] POM is updated
- [ ] A test is performed against the ACC environment from our local development environment to see if we can deploy several plans
- [ ] The plan publisher is executed on ACC
  - [ ] It should be able to deploy all plans
  - [ ] It should succeed running all unit tests
- [ ] It is checked if we can still communicate with the various Bamboo REST API's (See red remarks in the table above)
  - [ ] We add a simple test plan which can call another plan and read out the results to at least test the Bamboo API functionality
  - [ ]Code is adapted if needed
 - [ ] If any critical issue is found and which are not easy to solve, the responsible team is informed (and our colleagues in other System teams as well)
  - [ ]Inform David that we cannot test everything (due to the isolation - we cannot contact our VCP's)
  - [ ]The prepared PR is maintained (kept up to date), until the new Bamboo version is released to production
